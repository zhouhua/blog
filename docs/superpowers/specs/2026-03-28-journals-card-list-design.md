# 随笔列表页卡片化改版设计

日期：2026-03-28

## 背景

当前随笔列表页位于 `/journals`，最初采用时间轴形式组织内容。站点样式经过多轮重构后，该页面仍保留时间轴时期的结构和视觉元素，包括竖线、节点、横向连接线，以及依赖绝对定位的布局方式。页面虽然还能工作，但它已经不再匹配当前站点的视觉语言，也限制了后续维护。

这次改版不再修补时间轴，而是明确放弃时间轴表达，改为按年份分组的卡片列表。

## 目标

1. 保留按年份倒序分组的归档方式。
2. 将时间轴列表改造成真正的卡片列表设计。
3. 保留“在列表页直接阅读完整随笔内容”的体验，而不是改成摘要流。
4. 让页面视觉和首页、文章列表页当前风格更一致。
5. 提升布局稳定性，降低对绝对定位和装饰性伪元素的依赖。

## 非目标

1. 这次不加入搜索、筛选、年份折叠。
2. 这次不改成摘要卡或双列瀑布流。
3. 这次不增加 hover 动效。
4. 这次不重写单篇随笔页。
5. 这次不处理与卡片化无关的全站样式问题。

## 方案对比

### 方案 A：轻改旧结构

保留现有页面的主要 DOM 结构，只删除时间轴竖线、节点和连线，尽量用最少改动把页面变成“像卡片”的样子。

优点：
- 改动小。
- 风险低。

缺点：
- 容易残留时间轴布局的历史包袱。
- 卡片层次和信息组织会比较勉强。
- 结果更像“拆旧样式”，而不是“重做列表页”。

### 方案 B：重组为卡片列表

保留年份分组和完整正文渲染，但把每条随笔重构为明确的卡片单元：头部展示日期与阅读信息，主体承载正文内容，年份区块负责分组。

优点：
- 更符合本次“从时间轴切换到卡片列表”的目标。
- 信息结构更清晰。
- 与当前站点风格更一致。
- 后续扩展更容易。

缺点：
- 改动略大于轻改。
- 需要同步更新测试描述与断言。

### 结论

采用方案 B。

## 信息架构

页面结构调整为两层：

1. 顶部保留 `ListTitle`，继续承担页面标题与简介。
2. 主体内容区按年份倒序渲染多个区块。
3. 每个年份区块包含：
   - 年份标题
   - 该年份下的随笔卡片列表
4. 每条随笔卡片包含：
   - 卡片头部：日期、字数、预计阅读时间
   - 卡片主体：直接渲染 Markdown 内容

年份是分组信息，卡片是阅读单元，正文仍然是页面主角。

## 视觉方向

### 整体气质

目标是做成“整理过的随笔册”，而不是“信息密度很高的归档控制台”。视觉应保持安静、稳定、可连续阅读。

### 年份区块

- 年份标题保留，但从原先时间轴锚点式定位改成自然文档流中的区块标题。
- 年份标题应清晰但不过度抢眼，继续使用当前站点已有的等宽数字与强调色体系。
- 年份区块之间通过留白和轻量分隔建立节奏，不使用时间轴装饰线。

### 卡片外层

- 单列纵向排列，不做双列。
- 使用 `bg-card`、圆角和克制的阴影建立卡片边界。
- 卡片宽度铺满可用内容区，但维持适合阅读的最大宽度，避免超宽媒体撑坏布局。
- 卡片之间留出稳定的纵向间距，让页面可以连续阅读但不过于拥挤。

### 卡片头部

- 左侧为日期，如 `06-07`。
- 右侧为字数与阅读时间，视觉弱于日期。
- 头部与正文之间通过分隔线或留白过渡，不再使用绝对定位时间轴节点。
- 日期继续作为进入单篇页的入口。

### 卡片正文

- 继续直接渲染 `Content`，保留现有文章排版能力。
- 标题、引用、图片、列表等内容继续沿用 `ArticleBody` 的样式体系。
- 卡片正文需限制超宽内容，保证图片、代码块或嵌入内容不会破坏卡片布局。
- 内容开头自然进入正文，不再增加额外的时间轴装饰。

## 响应式行为

### 桌面端

- 每个年份下为单列卡片流。
- 卡片头部左右分布信息，元信息不压正文。
- 卡片内边距保持舒适阅读宽度。

### 平板与手机

- 继续单列，不切双列。
- 缩小卡片内边距，提高小屏有效阅读面积。
- 卡片头部在空间不足时允许换行，保证日期和元信息都清晰可见。
- 图片、引用、代码块需要自然收缩，避免横向溢出。

## 交互

1. 本次不加入 hover 动效。
2. 不增加折叠、展开、筛选等额外交互。
3. 日期链接保留，作为进入单篇页的明确入口。
4. 年份标题只承担分组作用，不吸顶、不折叠。

## 测试策略

测试文件仍保留两类验证，但命名和断言会调整：

1. 页面成功返回，且不存在历史 `toSorted` 兼容错误。
2. 页面仍然渲染年份分组和随笔卡片。
3. 卡片布局应包含新的结构特征，例如年份区块、卡片容器和正文容器。
4. 避免断言过于脆弱的视觉细节，例如具体阴影值或精确 spacing。

## 风险与缓解

### 风险 1：正文内容形态差异大

随笔有的很短，有的接近短文，还有图片、引用等富内容，卡片高度会自然变化。

缓解：
- 保持单列布局，不追求统一高度。
- 通过一致的头部结构和卡片间距维持整体节奏。

### 风险 2：旧时间轴样式残留

如果只是局部替换类名，可能会遗留伪元素或定位逻辑，影响新布局。

缓解：
- 直接删除时间轴相关样式和类名。
- 用新的年份区块与卡片结构取代旧布局。

### 风险 3：测试名称和语义落后于实现

如果测试仍以 timeline 命名，会让后续维护者误判页面设计意图。

缓解：
- 同步重命名测试描述。
- 使用“card list”或“journals cards”一类语义。
